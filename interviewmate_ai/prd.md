# InterviewMate AI PRD

## 1. 背景

转岗 AI 产品经理的候选人常见问题是：不知道岗位具体问什么、项目表达没有结构、缺少真实追问和复盘反馈。通用聊天机器人可以给题，但难以持续扮演面试官、追问细节并根据目标岗位生成训练计划。

InterviewMate AI 的目标是提供一个面向 AI PM 求职者的个性化面试陪练 Agent。

## 2. 目标用户

- 非科班转岗 AI PM 候选人。
- 应届生或低年限产品候选人。
- 有项目但不会讲 AI 产品闭环的求职者。

## 3. 用户痛点

| 痛点 | 表现 | 影响 |
| --- | --- | --- |
| 不知道问什么 | JD 看不懂，准备泛泛 | 命中率低 |
| 回答没结构 | 项目讲成流水账 | 面试官难判断能力 |
| 缺少追问 | 只背标准答案 | 一被深挖就崩 |
| 反馈不具体 | 不知道哪里弱 | 训练效率低 |

## 4. MVP 范围

### P0 功能

1. JD 解析
   - 提取岗位关键词、能力要求和业务方向。

2. 题库生成
   - 产品设计题、AI 技术理解题、项目深挖题、行为题。

3. 模拟面试
   - Agent 按开场、主问题、追问、纠偏、总结五阶段推进。

4. 回答评分
   - 产品结构、AI 理解、指标意识、表达可信度、岗位匹配度。

5. 训练计划
   - 根据薄弱项生成 7 天练习计划。

### P1 功能

- 语音模拟。
- 简历项目针对性追问。
- 多岗位版本对比。

## 5. Agent 流程

```text
解析 JD -> 选择面试模式 -> 提出主问题 -> 根据回答追问 -> 评分 -> 生成改进建议 -> 更新训练计划
```

## 6. 指标

### 产品指标

- 首次练习完成率。
- 7 日练习留存。
- 平均练习轮次。
- 用户自评信心提升。

### AI 质量指标

- JD 关键词提取准确率。
- 追问相关性。
- 评分一致性。
- 建议可执行性。

## 7. 风险控制

| 风险 | 策略 |
| --- | --- |
| 生成虚假经验 | 明确提示只基于用户真实经历包装表达 |
| 反馈过泛 | 使用评分 Rubric 和示例改写 |
| 追问跑偏 | 限定面试阶段和目标岗位 |
| 用户过度依赖 | 强调训练表达，不替代真实能力 |

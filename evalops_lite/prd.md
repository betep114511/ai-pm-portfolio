# EvalOps Lite PRD

## 1. 背景

LLM 应用上线前经常依赖产品经理、算法同学或业务方手动试几条样例，缺乏稳定的评测集和上线门禁。随着 Prompt、模型、知识库和工具调用频繁迭代，团队很难判断新版本是否真的变好，线上差评也无法沉淀成回归测试。

EvalOps Lite 的目标是为小型 AI 产品团队提供轻量评测工作台，让 AI 产品迭代从“主观感觉”变成“指标驱动”。

## 2. 目标用户

- AI 产品经理：定义指标、上线门禁和失败归因。
- Prompt 工程师：对比 Prompt 版本效果。
- 算法/工程团队：对比模型、检索参数和延迟成本。
- 业务审核/标注团队：人工抽检高风险样本。

## 3. 用户痛点

| 痛点 | 表现 | 影响 |
| --- | --- | --- |
| 评测不稳定 | 每次手测样例不同 | 无法判断版本优劣 |
| 指标不完整 | 只看准确率，不看风险和成本 | 上线决策片面 |
| 失败难归因 | 不知道是检索、模型还是 Prompt 问题 | 迭代效率低 |
| 线上问题不回流 | 差评样本没有进入测试集 | 同类错误反复出现 |
| 高风险缺人审 | LLM-as-Judge 直接替代人工 | 安全和合规风险 |

## 4. 产品目标

- 让团队能在 30 分钟内完成一次 Prompt/模型对比评测。
- 将线上差评样本沉淀为回归测试集。
- 为 AI 功能上线提供质量、风险、成本和延迟门禁。

## 5. MVP 范围

### P0 功能

1. 测试集管理
   - 导入问题、标准答案、场景标签、风险等级。
   - 支持样本分组：核心样本、高风险样本、线上回流样本。

2. 实验配置
   - 选择模型版本。
   - 选择 Prompt 版本。
   - 配置 RAG Top-K、温度、系统提示。

3. 自动评测
   - 规则检查。
   - LLM-as-Judge。
   - 格式遵守和引用检查。

4. 结果看板
   - 准确率、幻觉率、拒答正确率、引用命中率。
   - P95 延迟、单次成本。
   - 失败案例分布。

5. 上线门禁
   - 配置最低质量要求。
   - 高风险样本必须人工抽检。
   - 未达标版本禁止发布。

6. 失败回流
   - 线上差评一键进入回归测试集。
   - 标记失败类型：检索失败、模型幻觉、格式错误、拒答错误、安全违规。

### P1 功能

- 多人协作标注。
- 评测报告导出。
- 版本趋势追踪。
- 自定义 Judge Rubric。

### P2 功能

- 与线上日志系统集成。
- 自动采样灰度流量。
- 成本预算预警。

## 6. 核心流程

### 流程 A：Prompt A/B 评测

1. PM 选择一个测试集。
2. 配置 Prompt A 和 Prompt B。
3. 运行评测。
4. 看质量、风险、成本、延迟对比。
5. 决定是否灰度上线。

### 流程 B：线上失败回流

1. 用户对 AI 回答点差评。
2. 系统记录问题、上下文、回答、引用和模型版本。
3. PM 标记失败类型。
4. 样本加入回归测试集。
5. 下一次版本评测必须覆盖该样本。

### 流程 C：高风险人审

1. 高风险样本自动进入人工抽检队列。
2. 人审标记通过/不通过和原因。
3. 结果影响上线门禁。

## 7. 指标体系

### 北极星指标

每周通过门禁并完成灰度发布的 AI 功能版本数。

### 质量指标

- 准确率。
- 相关性。
- 完整性。
- 格式遵守率。
- 引用命中率。

### 风险指标

- 幻觉率。
- 安全违规率。
- 拒答正确率。
- 高风险样本通过率。
- 人工抽检不通过率。

### 体验与工程指标

- 任务完成率。
- P95 延迟。
- 单次 token 成本。
- 模型调用失败率。

## 8. 上线门禁示例

- 核心样本准确率 >= 85%。
- 高风险样本通过率 >= 95%。
- 幻觉率 <= 3%。
- 引用命中率 >= 85%。
- P95 延迟 <= 8 秒。
- 单次成本 <= 预算阈值。
- 高风险样本人工抽检完成率 = 100%。

## 9. 风险控制

| 风险 | 策略 |
| --- | --- |
| LLM-as-Judge 偏差 | 高风险样本人审，定期校准 Judge |
| 测试集污染 | 区分训练样本、开发样本、保留测试集 |
| 指标过度单一 | 质量、风险、体验、成本一起看 |
| 线上错误重复出现 | 差评样本回流回归集 |
| 成本失控 | 每次实验估算 token 成本 |

## 10. 项目边界

EvalOps Lite 不做模型训练平台，也不做完整 MLOps。它定位为 AI 应用团队的轻量评测和上线决策工具。

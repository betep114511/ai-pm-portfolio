# SafetyLens 用户研究

## 研究目标

验证实验室和 EHS 场景中是否存在 SDS 审核低效、危险信息遗漏和审核留痕不足的问题，并明确 AI 在流程中的合理边界。

## 用户画像

### Persona 1：EHS 专员

- 目标：确保危化品入库、储存和使用符合安全要求。
- 痛点：SDS 文件多、格式不统一，高风险项需要人工逐条确认。
- 关键需求：高风险召回、来源证据、审核记录。

### Persona 2：实验室管理员

- 目标：管理试剂库、废液和实验前安全检查。
- 痛点：新人不熟悉 PPE 和不相容物质，重复解释成本高。
- 关键需求：安全清单、储存建议、提醒机制。

### Persona 3：采购/仓储人员

- 目标：判断新化学品能否入库、应放在哪个库位。
- 痛点：不懂专业安全条款，依赖 EHS 反馈。
- 关键需求：初步风险分级、待确认事项、入库建议。

## 访谈问题

- 审核一份 SDS 通常需要多长时间？
- 最容易遗漏哪些字段：PPE、储存条件、不相容物、急救措施？
- 哪些判断必须由 EHS 人工确认？
- 现在是否有完整审计记录？
- 如果 AI 先给出抽取结果，你愿意在哪些环节使用？

## 需求洞察

### 洞察 1：安全场景更怕漏报而不是误报

误报会降低效率，但漏报可能导致事故。因此 MVP 指标应优先保证高风险召回率。

### 洞察 2：AI 适合做“初审”，不适合做“最终责任人”

用户愿意让 AI 整理字段、提取证据和生成清单，但高风险结论仍需要 EHS 签字。

### 洞察 3：审计留痕是刚需

合规场景需要知道 AI 原始输出是什么、谁改了什么、最终依据是什么。

## MVP 优先级

| 需求 | 用户价值 | 优先级 |
| --- | --- | --- |
| SDS 字段抽取 | 高 | P0 |
| 高风险规则校验 | 高 | P0 |
| 人工审核状态 | 高 | P0 |
| 审计日志 | 高 | P0 |
| 批量审核 | 中 | P1 |
| 库位推荐 | 中 | P1 |

# SafetyLens Case Study

## 项目概述

SafetyLens 是一个危化品 SDS 安全合规审核助手，面向实验室管理员、EHS 专员和化工仓储人员。它通过 LLM 抽取 SDS 字段，并结合规则引擎识别危险类别、PPE、储存条件和不相容物质，最后进入人工审核和审计留痕。

## 背景问题

SDS 文件通常篇幅长、格式不统一，而且安全信息分散在多个章节。人工审核耗时高，新人容易漏看危险说明、储存条件和急救措施。AI 可以帮助提取和总结，但安全合规场景不能让模型独立做最终判断。

核心问题：

- 如何提升 SDS 审核效率。
- 如何避免高风险漏报。
- 如何保证 AI 输出可追溯、可审核。

## 目标用户

- EHS 专员：关注高风险项、合规记录和审核责任。
- 实验室管理员：关注储存、PPE、废液和培训。
- 仓储/采购人员：关注新化学品是否可入库。

## 解决方案

SafetyLens 的设计是“LLM 抽取 + 规则校验 + 人工确认”：

1. LLM 从 SDS 中抽取化学品名称、CAS、GHS 分类、PPE、储存条件。
2. 规则引擎检查易燃、氧化、腐蚀、毒性和不相容物质风险。
3. 高风险项必须由 EHS 人工确认。
4. 系统记录 AI 原始输出、人工修改和最终审核状态。
5. 生成安全检查清单，用于入库或实验前确认。

## MVP 范围

- SDS 上传。
- 字段抽取。
- 风险摘要。
- 不相容规则校验。
- 高风险人工审核。
- 审计日志。
- 安全 checklist 导出。

## 关键产品决策

### 决策 1：不让 LLM 做最终合规结论

LLM 更适合抽取和总结，明确安全规则由规则引擎处理，高风险判断由人工确认。

### 决策 2：按风险分层展示

低风险信息可直接展示，高风险项进入待确认队列，避免用户被大量警报淹没。

### 决策 3：审计留痕是核心功能

合规场景里，谁改了什么、依据是什么，比回答是否漂亮更重要。

## 指标体系

北极星指标：

- 每周完成并通过人工确认的高风险 SDS 审核数。

质量指标：

- CAS/名称抽取准确率。
- 高风险项召回率。
- 不相容物质召回率。
- 高风险漏报率。
- 人工修改率。

业务指标：

- 单份 SDS 初审耗时下降。
- 人工审核完成率。
- 清单导出次数。

## 风险控制

| 风险 | 设计 |
| --- | --- |
| 抽取错字段 | 字段级置信度和人工修改 |
| 高风险漏报 | 规则引擎兜底，高风险召回优先 |
| 误报过多 | 风险分层、人工标记、规则迭代 |
| 责任不清 | 审计日志保留 AI 和人工版本 |
| SDS 过期 | 版本和发布日期提醒 |

## 面试总结

> SafetyLens 体现的是我对高风险 AI 产品的理解：AI 不应该替代合规责任人，而是帮助整理证据、提高初审效率，并通过规则、人审和审计留痕降低风险。
